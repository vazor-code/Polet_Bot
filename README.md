# OC_POLET_BOT

Max-бот для регистрации пользователей образовательного центра и хранения профилей в SQLite.

## Возможности

- Регистрация пользователя через диалог (`/register`)
- Проверка и нормализация номера телефона
- Сохранение профиля в локальную базу `SQLite`
- Просмотр профиля (`/profile`)
- Отмена текущей регистрации (`/cancel`)

## Стек

- Python 3.10+
- `maxapi`
- SQLite (встроен в Python)

## Установка

1. Клонируй репозиторий и перейди в папку проекта:

```bash
git clone <URL_репозитория>
cd OC_POLET_BOT
```

2. Создай и активируй виртуальное окружение:

Windows (PowerShell):

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

macOS/Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

3. Установи зависимости:

```bash
pip install maxapi
```

## Настройка

Боту нужен токен в переменной окружения `MAX_BOT_TOKEN`.

Windows (PowerShell, на текущую сессию):

```powershell
$env:MAX_BOT_TOKEN="твой_токен"
```

macOS/Linux:

```bash
export MAX_BOT_TOKEN="твой_токен"
```

Если переменная не задана, бот завершится с ошибкой при старте.

## Запуск

```bash
python bot.py
```

При первом запуске создается/обновляется база `school_bot.db` в корне проекта.

## Команды бота

- `/start` — главное меню
- `/help` — список команд
- `/info` — информация о боте
- `/register` — регистрация пользователя
- `/profile` — показать профиль
- `/cancel` — отменить регистрацию

## Структура данных

Таблица `users`:

- `max_user_id` — ID пользователя в MAX (уникальный)
- `phone` — телефон (уникальный, в формате `+7XXXXXXXXXX`)
- `full_name` — ФИО
- `role` — роль пользователя
- `created_at`, `updated_at` — служебные даты

## Полезно знать

- Файл базы данных: `school_bot.db`
- Логи выводятся в консоль с уровнем `INFO`
- Повторный `/register` обновляет данные пользователя

## Возможные проблемы

- `RuntimeError` про токен: проверь `MAX_BOT_TOKEN`
- Ошибка импорта `maxapi`: выполни `pip install maxapi`
- Бот не отвечает: проверь корректность токена и доступ к сети
